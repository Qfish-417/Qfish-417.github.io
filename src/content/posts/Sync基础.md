#  5.1 除了 mutex 以外还有那些方式安全读写共享变量？

主要还有**信号量**、**通道（Channel），原子操作（atomic）**这几种方式。

> 除了 Mutex 之外，还有几种并发安全方式：
>
> - **Semaphore（信号量）**：通过计数控制同时进入临界区的 goroutine 数量；适合限制并发。
> - **Channel**：利用 CSP 模型用消息代替共享内存，本质是无锁同步，天然保证 goroutine 间通信安全。
> - **Atomic 原子操作**：基于 CPU 指令的无锁操作，适用于简单数值的安全读写，如计数器和状态位。

# 5.2 Go 语言是如何实现原子操作的？

Go 的原子操作是通过 CPU 原子的硬件指令实现的，同时结合汇编实现和内存屏障来保证原子性、顺序性和可见性，从而实现无锁并发。

# 5.3 原子操作和锁的区别（面试高频回答）

原子操作是依赖 CPU 指令实现的无锁机制，适合单变量的高性能更新；而锁通过阻塞/唤醒实现互斥，适合保护复杂的临界区。原子操作快但难写，锁更通用但代价更高。

# 5.4 Go语言互斥锁mutex底层是怎么实现的？

Go 的 mutex 用一个状态字段实现“自旋 + 阻塞”的混合模式，并在激烈竞争时自动切换到饥饿模式保证公平；加锁通过 CAS 和自旋快速路径实现，失败才进入调度器阻塞队列；解锁时根据模式选择是否直接交接锁或让竞争者抢锁。

# 5.5 Mutex 有几种模式？

o 的 Mutex 有两种模式：

- **正常模式**：CAS + 自旋，性能最好，但有可能饿死某些 goroutine
- **饥饿模式**：排队公平依次获取，不会饿死，但性能稍低
   Mutex 会根据竞争情况在两种模式之间自动切换。
